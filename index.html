Project Description

I am the troll runner. I troll around the world one mile at a time. While I do this I put stickers around the world. The sticker is basically a picture of the iconic troll face meme and just below it, is the crypto IP trollface project's website, trololol.app. I'm a long distance runner. I have placed about 80 stickers around Southern California.

Design Philosophy

  • Vibe: Modern, colorful, playful, and mischievous
  • Colors: Bold, high-contrast palette with bright primary color for actions
  • Typography: Distinctive, legible sans-serif with quirky display fonts
  • Imagery: Troll face sticker motif, map/location visuals, travel and exploration themes
  • Layout: Scroll-based with sections: hero, biography, map/locations, social links
  • Tone: Fun, irreverent, adventurous - but clean and navigable

🎨 Customization Guide

Design Tokens (Colors, Typography, Spacing)

Edit these files to customize the site's appearance:

  • tailwind.config.js - Primary design tokens (colors, fonts, sizing)
  • src/styles/global.css - Global CSS variables and Tailwind layer overrides

Adding New Pages

  1. Create a new .astro file in src/pages/
  2. Import Header.astro and Footer.astro
  3. Add the page content between them
  4. Include RequiredB12HeadScripts and RequiredB12FooterScripts

Adding Interactive Components

Use React components with Astro's Islands directive:

--- import MyComponent from '../components/MyComponent'; --- <MyComponent client:load />

Updating Links & Content

  • Edit the homepage content in src/pages/index.astro
  • Update social media links in the relevant components
  • All colors and styling use Tailwind utility classes
  • Forms use B12's integrated form system with reCAPTCHA protection

⚠️ Important Notes

Required B12 Components: The RequiredB12HeadScripts and RequiredB12FooterScripts components must remain on every page. Do NOT move, remove, or modify them. They handle critical B12 platform functionality.
B12Link Component: Every page footer must contain exactly one <B12Link /> component. This is required for B12 attribution.
Section Navigation: All major sections must have an id attribute (slugified). For example: id="hero", id="about", id="contact". This enables URL fragment navigation.
Responsive Design: Always test designs on mobile. Use Tailwind's responsive prefixes: sm:, md:, lg:, xl:, 2xl:
TypeScript: The project uses TypeScript for type safety. Always use proper types for React components and configuration files.
